
Trials and tribulations to install and test Apache on CentOS6.6

~$ su
~$ yum install epel-release
~$ yum install httpd (maybe already installed)
~$ yum install mod_wsgi

a. Test with sample Django project: remember to serve on 0.0.0.0:xxxx for external access (firewall should already be disabled)
b. Configure apache
  ~$ sudo vim /etc/httpd/conf.d/django.conf
    # Allow apache to directly access static directory
    Alias /static /home/boo/code/django/test_apache/static
    <Directory /home/boo/code/django/test_apache/static>
        Require all granted
    </Directory>

    # Define root for wsgi
    <Directory /home/boo/code/django/test_apache/test_apache>
        <Files wsgi.py>
            Require all granted
        </Files>
    </Directory>

    # wsgi processes
    WSGIDaemonProcess test_apache python-path=/home/boo/test_apache:/home/boo/.virtualenvs/apache_test/lib/python3.4/site-packages
    WSGIProcessGroup test_apache
    WSGIScriptAlias / /home/boo/test_apache/test_apache/wsgi.py

  ~$ sudo usermod -a -G boo apache (create user 'apache' within same group as 'boo')
  ~$ chmod 710 /home/boo (give apache user access to home directory)
  ~$ chmod 664 /home/boo/code/django/test_apache/db.sqlite3 (Only if using sqlite)
  ~$ sudo chown :apache /home/boo/code/django/test_apache/db.sqlite3 (give apache group onwership of db file)
  ~$ sudo chown :apache /home/boo/code/django/test_apache (Ownership of db parent directory)

c. Run apache: ~$ apachectl start
d. For status
  i. ~$ yum install links
  ii. ~$ apachectl status
  iii.
d. Run apache at startup: ~$
